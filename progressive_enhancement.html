What is Progressive Enhancement?

  • "Progressive enhancement is a strategy for web design that emphasizes accessibility, semantic markup, and external style-sheet and scripting technologies." Wikipedia
  • Coined by Steven Champeon for Webmonkey and the SXSW Interactive conference in 2003 Wikipedia

Why should I?

What you get for free

  • Compatibility with everything that supports HTML
  • Your code will make more sense to more people
  • Better compatibility with future platforms

Freedom to move ahead

  • Not much has changed much in JavaScript or HTML since 1999
  • Fear of "breaking the web" is a top reason
  • This does not need to be an issue

It's not more work, it's a different way of looking at your development

  • Strip down the metaphors. Do you remember the Nob control?
  • What is the standard HTML supported way of accomplishing the same goal
  • Question: What is the standard HTML supported way of accomplishing the drag and drop interface in Basecamp?

Where does Ruby on Rails come in?

  • Simplicity of creating HTML and JSON views from one controller?
  • Adherence to REST interfaces

There are lots of different ways to do this

  • Some sites can totally degrade and some cannot
  • Some features are not needed at all. Curves maybe?
  • Information sites should make all information available to every user
  • Web Apps should make sure that there is at least end to end functionality
